Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with AWS, setting up Console/CLI and learning EC2 basics](1.1-week1/)

**Week 2:** [Learning Amazon VPC architecture, Subnet, Route Table, Security Group and Load Balancer](1.2-week2/)

**Week 3:** [Researching Amazon EC2, EBS, Instance Store, Auto Scaling and Pricing Options](1.3-week3/)

**Week 4:** [Learning Amazon S3, Storage Class, Lifecycle Policy, Glacier and Disaster Recovery](1.4-week4/)

**Week 5:** [Researching IAM, Shared Responsibility Model, Cognito and AWS Security Services](1.5-week5/)

**Week 6:** [Learning Database Concept, RDBMS/NoSQL, RDS, Aurora, Redshift and ElastiCache](1.6-week6/)

**Week 7:** [Setting up HR System project: AWS IAM, Spring Boot, S3 Static Website Hosting](1.7-week7/)

**Week 8:** [Deploying CloudFront CDN, Route 53 DNS and AWS WAF for security](1.8-week8/)

**Week 9:** [Configuring API Gateway, Cognito User Pool and JWT Authorizer authentication](1.9-week9/)

**Week 10:** [Developing Employee and Payroll APIs with Spring Data JPA and Repository](1.10-week10/)

**Week 11:** [Implementing Attendance API, Lambda Flow and Leave Management System](1.11-week11/)

**Week 12:** [Optimizing with Redis Cache, Secrets Manager and completing API documentation](1.12-week12/)
